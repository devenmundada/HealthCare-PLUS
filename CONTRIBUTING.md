# Contributing

## Getting Started

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Run the app locally and verify everything works
5. Commit: `git commit -m "feat: describe your change"`
6. Push and open a Pull Request

## Commit Format

Use conventional commits:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `refactor:` code change, no feature/fix
- `test:` adding tests

## Code Standards

- TypeScript for all new frontend code
- No `any` types unless absolutely necessary
- Keep components small and focused
- Add JSDoc comments for non-obvious functions

## Reporting Issues

Use the GitHub issue templates. Include steps to reproduce, expected vs actual behavior, and your environment (OS, Node version, browser).
